Exercise 1

We are playing blackjack.

Create an enumeration type CardFace that has the values TWO, THREE etc.

  • Each card has a value as follows.
    • The value of cards two through ten is their pip value (2 through 10).
    • Face cards (Jack, Queen, and King) are all worth ten.
    • Aces are always worth eleven. (This is a simplification of the real rules.)

In the class Main1, create main that does the following computation.

  • Note that you must do the computation using a single Stream, otherwise your solution will not be accepted.
    • For example, if you try to use a loop, your solution is worthless.
    • You may open the standard input before.
  • The standard input will contain words that are names of card faces or the word done.
    • Example: FIVE TEN QUEEN JACK done
  • Sum the values of all except, but leave out all repeated cards.
    • For this, you will have to maintain a state.
  • Once you see the word done, you’re finished.
    • Print the resulting sum.

You must solve this exercise fully before moving on to other exercises.

Present your solution to me when you’re done.

Exercise 2A

In class Main2A, create a server on port 10000.

When a client connects to the server, he sends in card names like in Exercise 1, followed by done. Let the server answer with the card value sum (as produced in Exercise 1), then the server disconnects from the client.

Also, allow the server to handle many connections in parallel.

Exercise 2B

Create the annotation CardPlayer. Make it so that methods can be annotated with it. It has two fields: a player name and a CardFace.

Create the class CardTournament. In it, annotate some (static) methods with CardPlayer; the methods will take no arguments, and return an int.

In the class Main2B, write a program that decides the tournament winner, that is, the player with the highest score.

  • The score is computed like this: for all players, sum the card values from all the annotated methods that have the player’s name, except where the player is cheating.
  • To decide if player is cheating on a card, call the method on which the annotation is present. The card is cheated if the return value of the method and the card’s value are different, or if the method’s name contains cheat textually.

Exercise 2C

In class Main2C, open the file java.exe using NIO. Read bytes from the indexes 1, 3, 6, 10, 15, 21 etc. up to the end of the file.

If there is a card whose value is exactly that of a card, print the card’s name. For 10, print KING.
